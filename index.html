Rating System Automation

This project is a automation project for Habertürk. The purpose of this project is to show the rating data of television channels received every day. Rating data comes with email and needs to be processed before being shown to the user. This project consists of 4 parts: Console Application, Database, API, and UI. The console Application gets the data from email and inserts it into the Microsoft SQL database. Data is processed through certain algorithms using the procedure method in the database. When the ASP.NET API is called it fetches the data from the database. This API sorts and colors the data to prepare for the demonstration. Finally, the Angular web application displays the data in various tables.

Reader-Writer Synchronization

This project implements a semaphore-based solution to the Readers-Writers Problem, ensuring controlled access to shared variables. Readers can access data simultaneously, while writers require exclusive access to maintain consistency. By using semaphores, the solution prevents race conditions and optimizes performance. Through this project, I strengthened my understanding of multithreading, synchronization, and concurrent programming, key concepts in operating systems.
